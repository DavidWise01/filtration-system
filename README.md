# THE FILTRATION SYSTEM

> A sphere of **UD0** — the ROOT0 universe / biosphere. Domain: **ESKIMO BROTHERS**.

THE FILTRATION SYSTEM — built ON [[duality-engine]] (imports its MACI validator): a tiered water-filter for raw multi-party logs that answers WHAT DID WE ACTUALLY DECIDE, AND WHO HAD THE AUTHORITY. Three filters, each only REMOVES or DEMOTES (never adds back), so what survives is high-confidence by construction and every removal lands in an auditable tray. FS (coarse screen): mechanical parse only — who/when/literal content + authority from the directory; drops non-messages & unknown speakers; can't be 'wrong'. FSS (forward, optimistic): generous role promotion tuned for ZERO false negatives — if a line could be a decision, tag it (over-tag on purpose). BSS (backward, skeptical): demotes the unbacked — a decision by a non-authority, an off-topic red herring, a casual non-commitment. The FSS/BSS GAP = the contested set routed to human review; what survives both = CERTIFIED. Worked example: a 20-message Slack thread choosing a cache — the contractor's UNAUTHORIZED 'im switching us to memcached' (the change that caused the 2:30 incident) is demoted (advisory cannot decide) and the certified answer is what the SOVEREIGN actually called: cache = REDIS. LIVE in-browser (the ported pipeline + an authority-blind toggle showing what an unfiltered reader would wrongly believe). Mirrors the engine's converged FSS/BSS auditor, pointed at real-world provenance. Ships tiered_extractor.py + the worked example. Honest: role cues are regex heuristics, not language understanding — hence the FS foundation + the human-review tray; built to be auditable, not blindly trusted. David Lee Wise / Bridge-Burners LLC.
